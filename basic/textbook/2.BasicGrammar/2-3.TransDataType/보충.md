# Golang 데이터 타입 변환 & 관련 개념 정리

## 1. `strconv`란?
- Go 표준 라이브러리(`package`)
- 이름: **String Convert**
- 문자열 ↔ 숫자 변환 담당
  - `Atoi` : 문자열 → int
  - `Itoa` : int → 문자열
  - `ParseInt`, `ParseFloat`, `ParseBool`
  - `FormatInt`, `FormatFloat` 등

---

## 2. `Atoi` vs `ParseInt`
### `strconv.Atoi`
- "ASCII to Integer"
- 문자열을 **10진수 int**로 변환
- 내부적으로 `ParseInt(s, 10, 0)`을 호출 (base=10, bitsize=int)

### `strconv.ParseInt`
- 더 범용적
- `(문자열, base, bitsize)` 형태로 사용
  - `base`: 진법 (2, 8, 10, 16…)
  - `bitsize`: 0, 8, 16, 32, 64 → 범위 검사
- 반환 타입: `int64`

### 차이 정리
- `Atoi` = 간단하게 문자열 → int
- `ParseInt` = 진법/비트 크기까지 직접 지정 가능

---

## 3. `err`란?
- Go 함수는 보통 `(결과, error)` 형태 반환
- `err`는 **에러 정보**를 담는 내장 타입(`error`)
- 성공 → `err = nil`
- 실패 → `err`에 오류 메시지나 범위 초과 정보 저장

### 예시
```go
i, err := strconv.Atoi("123")   // 성공
// i=123, err=nil

i, err := strconv.Atoi("abc")   // 실패
// i=0, err="invalid syntax"
````

---

## 4. ParseInt에서 `err`가 필요한 이유

* 변환 과정에서 두 가지 문제 가능

  1. **잘못된 입력**: `"XYZ"` → invalid syntax
  2. **범위 초과**: `"200"`을 int8로 변환 → 127, `ErrRange`

* 따라서 `err`는 “오차 보정”이 아니라
  👉 변환이 **정상인지 실패했는지 알려주는 보고서**

---

## 5. `reflect`란?

* Go 표준 라이브러리 `reflect`
* **Reflection(반사)**: 프로그램이 실행 중 자기 자신의 타입/값을 조사 가능
* 주요 함수:

  * `reflect.TypeOf(x)` → 변수의 타입 확인
  * `reflect.ValueOf(x)` → 값을 `reflect.Value`로 감싸서 세부 정보 확인

### 예시

```go
x := 42
fmt.Println(reflect.TypeOf(x)) // int
```

### 활용

* 일반 코딩에서는 거의 안 씀
* **라이브러리/프레임워크** 개발 시 유용

  * JSON 파싱
  * ORM (DB ↔ Struct 매핑)
  * 테스트 프레임워크 등
